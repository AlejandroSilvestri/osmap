#define FEATURES_MESSAGE_LIMIT   1000000

FEATURES_MESSAGE_LIMIT is the maximum number of features allowed in a single protocol buffer's message, to avoid known size problems in protocol buffers.

When a map exceed this limit, mapSave will save features file in delimited form, sequencing many messages each of them below this limit. This constant can be defined elsewhere before this point, or else is defined here.
